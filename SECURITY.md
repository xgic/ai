# Security Policy

## Supported content

This repository contains documentation and standards only. It does not ship runtime services.

## Reporting a vulnerability or information exposure

If you believe this repository contains **secrets**, **credentials**, or **private coordination details** that should not be public:

1. Do **not** open a public issue with the sensitive content.
2. Contact the maintainers privately via the GitHub Security Advisory workflow for this repository (if enabled) or the organization contact associated with [github.com/xgic](https://github.com/xgic).
3. For general documentation mistakes that are not sensitive, open a normal issue or pull request.

## Public hard-security expectation

XGIC public repositories must not document private infrastructure, private repository inventories, or private tracker identifiers. See [docs/BASE-STANDARDS-FOR-ORCHESTRATED-REPOS.md](docs/BASE-STANDARDS-FOR-ORCHESTRATED-REPOS.md).
